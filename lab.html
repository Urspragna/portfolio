AI Learning Lab.

Four hands-on demos that show what's actually happening inside an AI app — RAG retrieval, embeddings, sentiment classification, and tokenisation. Every demo is wired to the FastAPI backend at /api/*. Open /api/docs to see the full OpenAPI schema.

// what just happened

  1. Chunk. The CV is split into semantic units at ## CHUNK: markers — see data/knowledge_base.md.
  2. Embed. Each chunk + your query becomes a 384-dim vector via sentence-transformers/all-MiniLM-L6-v2.
  3. Retrieve. Cosine similarity ranks chunks; top-k goes into the prompt.
  4. Augment + Generate. A system prompt instructs the LLM to answer only from context — that's what makes RAG faithful instead of hallucinatory.
// demo 02 — Vector embeddings

How similar are two strings?

Embeddings turn text into a high-dimensional vector. Similar meanings → vectors close together. Cosine similarity captures the angle between them: 1.0 = identical direction, 0.0 = orthogonal, -1.0 = opposite.

// result

Run the demo to see vector similarity.

// the intuition

Modern LLMs operate on embeddings under the hood — every token, every chunk, every search query becomes a vector. RAG is just embedding similarity at scale. Try synonyms, antonyms, or text in different languages and watch the score change.
